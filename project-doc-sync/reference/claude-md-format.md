# CLAUDE.md 更新规范

本文件定义 `CLAUDE.md` 更新时的内容标准和章节定位规则。

---

## 章节定位策略

CLAUDE.md 的章节结构因项目而异。更新时应先读取当前 CLAUDE.md 的目录结构，按以下通用分类匹配章节：

| 内容类别 | 典型章节名 | 更新场景 |
|---------|-----------|---------|
| 项目概述 | "概述" / "Overview" | 技术栈变更、项目定位变更 |
| 架构决策 | "架构" / "Architecture" / "核心决策" | 新增/修改架构方案 |
| 状态机 / 业务流程 | "状态机" / "流程" / "State Machine" | 状态枚举变更、转换规则变更 |
| API 设计 | "API" / "接口" / "Endpoints" | 新增/修改 API 端点 |
| 第三方集成 | "集成" / "Integration" / "对接" | 外部平台接口变更 |
| 项目结构 | "结构" / "Structure" / "目录" | 新增/删除/移动文件 |
| 开发规范 | "规范" / "Convention" / "约定" | 新增代码约定 |
| 启动指南 | "启动" / "Quick Start" / "Getting Started" | 启动参数/步骤变更 |
| 设计说明 | "设计" / "Design" / "决策" | 新增设计决策说明 |
| 故障排查 | "排查" / "Troubleshooting" / "FAQ" | 新增排查经验 |

**定位流程：**
1. 读取当前 CLAUDE.md 的一级/二级标题列表
2. 按上表分类匹配最相关的章节
3. 若无匹配章节，追加到文末或"设计说明"类兜底章节

---

## 更新原则

### 1. 最小化变更

- 只改动与当前讨论直接相关的章节
- 不做全文重排、不改无关章节的措辞
- 保持现有的 Markdown 格式风格（表格对齐、代码块语言标注、缩进层级）

### 2. 新增内容的定位规则

| 变更类型 | 放在哪里 |
|---------|---------|
| 新增 API 端点 | API 设计相关章节 |
| 新增 DTO / Entity / Model | 项目结构章节 → 对应目录 |
| 新增配置项 | 对应功能模块的章节 |
| 新增状态 / 枚举 | 状态机 / 业务流程章节 |
| 新增文件 | 项目结构章节 → 目录树 |
| 新增设计决策 | 设计说明章节（新增子标题） |
| 新增代码规范 | 开发规范章节 |
| 新增故障排查 | 故障排查章节 |

### 3. 设计说明的格式

新增设计说明使用三级标题，格式：

```markdown
### {决策名称}

**问题**：描述需要解决的问题

**方案**：描述选择的方案

{代码示例（如有）}

**覆盖场景**：
- 场景 1
- 场景 2

**不影响的场景**：场景说明（如有）
```

### 4. 代码规范的格式

新增代码规范使用加粗前缀 + 说明：

```markdown
- **统一使用 `XxxUtil.method()`**，不要在各处重复实现 xxx 逻辑
```

### 5. 项目结构的格式

新增文件使用树形格式 + 注释：

```markdown
│   ├── NewFile.java               # 一句话说明文件职责
```

---

## 不应更新 CLAUDE.md 的场景

- 纯粹的 bug 修复（不涉及设计变更）
- 变量重命名、import 整理
- 测试代码的修改
- 注释的修改
- 不影响架构的局部重构
