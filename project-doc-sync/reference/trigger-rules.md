# 触发判定规则

本文件定义了 project-doc-sync 的触发判定细则，帮助 Claude 准确判断何时需要同步文档。

---

## todo.md 追加触发词

对话中出现以下语义模式时，应追加 TODO：

### 强触发（直接执行）

- "这个我们后面做" / "先记下来"
- "加到 todo" / "记到待办"
- "这需要改" / "这得修"（讨论后达成共识时）
- 讨论出明确的任务清单（连续列举 2 个以上待办项）
- 代码审查中发现问题并确认需要修复

### 弱触发（需用户确认再执行）

- "这个可以优化" / "以后可以改进"（建议性质，非明确任务）
- 提到了潜在问题但未明确是否要修

## todo.md 删除触发词

### 强触发（直接执行）

- 代码已实现 + 验证通过（编译成功/测试通过）
- 用户明确说 "这个做完了" / "已经解决" / "可以划掉了"
- 代码审查中确认功能已存在

### 弱触发（确认后执行）

- 功能代码已写但未验证
- 用户说 "应该可以了"（不够确定）

---

## CLAUDE.md 更新触发词

### 强触发（直接执行）

- 新增了 Controller、Service、Entity、DTO 文件
- 新增了 API 端点
- 修改了状态机转换规则
- 新增了配置项（application.yml / config.yaml）
- 讨论并确认了新的架构方案
- 新增了开发规范/代码约定
- 新增了故障排查经验

### 不触发

- 修改了现有代码但不改变设计（bug 修复、性能优化）
- 只修改了测试代码
- 只修改了注释
- 只做了代码格式化

---

## 关联文档级联更新触发词

### 强触发（直接执行）

当 CLAUDE.md 更新触发后，以下场景自动级联更新关联文档：

- CLAUDE.md 中引用了 `docs/xxx.md`，且本次变更直接涉及该文档描述的模块
- 修改了已知问题（CLAUDE.md 引用的 `docs/known-issues.md` 有对应条目）
- 新增/修改了 API 端点，且存在独立的 API 规格文档被 CLAUDE.md 引用
- 确认了新的架构方案，且存在独立的设计文档被 CLAUDE.md 引用
- CLAUDE.md 中新增了对某文档的引用（`详见 docs/xxx.md`），需确保被引用文档中有对应内容

### 弱触发（确认后执行）

- 关联文档内容与本次变更可能相关，但不确定是否需要更新
- 关联文档的变更范围较大，需要用户确认

### 不触发

- CLAUDE.md 未被更新时，不触发关联文档扫描
- 关联文档描述的模块/功能与本次变更完全无关
- 被引用的是外部链接（非项目内文件）
- 被引用文件不存在
